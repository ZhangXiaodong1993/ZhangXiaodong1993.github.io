麻雀虽小，五脏也全

发表于 2016-05-04   |   分类于 技术   |     |   阅读次数

花了2晚的时间终于把博客较为完善的搭建起来了。

参考链接

  • Hexo+Next部署
  • Hexo文档
  • Next文档(用处很大)
  • MarkDown语法简要版

里面的坑

  • Hexo安装指令前 + sudo，不然会说没权限
  • 部署到github，域名与github账号名要一致
  • 站点配置文件，url不要动，不要学第一篇文中所写，不要js会失效而报错
  • 统计阅读量，创建class需要全部权限，其页面已改版，跟文中不一致
  • 文件文件夹里面不要为空，不要也会导致js失效报错
  • 图片上传，markdown自带的格式并不好用，需要用到hexo文档中介绍的格式。

fi表示图片突出显示（会变大），Next文档有介绍。其它样式参考hexo的文档。上传的图片尺寸不要太大，最好控制在1M以内（如果很大，自行压缩一下），这样打开的时候不会加载很慢。

感谢以上作者，为我提供的所有帮助。

还有的问题

  • 目前搜索功能不好用(以去除，个人感觉没什么用)
  • 若要评论，登录之后需要刷新页面
  • 没有单独分类的功能，例如我想在最上部分设置一个按钮，是单独的一个分类，例如技术，点击显示所有的技术文章。目前没见到有这样使用的，如 小土刀博客 ，虽然有此分类，但明显是markdown写的文章索引。想实现的是类似 ibireme博客 的单独分类功能。(目前已得知是用WordPress搭建的)正在寻求方法，或者说换一个博客引擎。
